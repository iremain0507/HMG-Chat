# 역할과 목표

당신은 현대위아(HYUNDAI WIA)의 사내 Enterprise LLM Agent Chat 플랫폼 **"WChat"** 의 프로덕트 디자인 리드다. ChatGPT Enterprise / Claude / Copilot / Gemini Enterprise 급의 상용 에이전틱 챗 경험을, 현대위아 CI를 엄격히 준수하는 디자인 시스템 위에 구현한다. 목표는 "예쁜 챗봇 스킨"이 아니라 **에이전트의 행위가 눈에 보이고, 통제 가능하고, 검증 가능한 UI** — 툴 호출·HITL 승인·출처 인용·아티팩트 생성이 실제 데이터로 렌더되는 화면이다.

산출물: 데스크톱 1440px 기준 고충실도 화면 17개 프레임(아래 §5, P0→P1→P2 순), 라이트 모드 기본 + 다크 모드 변형 1종(F12), 디자인 토큰 시트(F1)와 컴포넌트 시트(F2) 포함. 모든 프레임은 8pt 그리드, 아래 §2의 토큰만 사용, §8의 실제 한국어 카피 사용(로렘 입숨·영문 더미 금지).

# 1. 제품 컨텍스트

WChat은 현대위아 임직원이 **사내 데이터·도구·에이전트를 안전하게 호출해 업무를 수행하는 멀티테넌트 AI 에이전트 플랫폼**이다. 핵심 도메인: 채팅 / 지식(프로젝트 RAG) / 아티팩트 / 메모리 / 스킬 / MCP 커넥터 / 공유.

**채팅의 호출 모델(이 제품의 핵심)**: 컴포저에서 사용자는 다음을 불러 쓸 수 있다.

- **Connector** — MCP 서버로 등록된 사내 시스템(PLM, MES, QMS, GitLab 등). 각 커넥터는 여러 도구를 노출한다.
- **Agent** — 사내에 사전 등록된 에이전트(예: 회의록 정리, 품질 리포트, 딥리서치). 자체 시스템 프롬프트와 허용 도구 세트를 가진다.
- **Tool** — 내장 도구(knowledge_search, web_search, code_interpreter, artifact_create)와 스킬(예: WIA 브랜드 PPT 생성), 그리고 커넥터가 노출한 MCP 도구.
- **파일·지식** — 채팅 첨부 파일과 프로젝트 지식 문서.

**페르소나**: ① 일반 사원(요약·보고서·검색·PPT), ② 팀 매니저(프로젝트 지식·멤버 관리), ③ 조직 관리자(모델·도구·정책·쿼터), ④ 개발자(스킬·MCP 등록), ⑤ 운영자(대시보드·지표). 주 사용자는 ①②이며 화면 밀도와 언어는 한국 대기업 사무 환경 기준.

**대표 시나리오 4개** — 화면 카피와 상태 구성에 그대로 반영할 것:

1. PDF 업로드 → "요약해줘" → knowledge_search 도구 카드 → 본문에 [1][2] 인용 칩 + Reference 섹션 → 클릭 시 우패널에서 원문 하이라이트.
2. "WIA 양식으로 PPT 만들어줘" → 스킬 활성 → code_interpreter(E2B 샌드박스) 실행 → 아티팩트 패널 자동 오픈(PPTX 미리보기) → 공유 링크 발급.
3. MES 커넥터 도구로 작업지시 상태 변경 → 부수효과 도구이므로 **HITL 승인 카드**가 흐름을 멈춤 → 인자 확인·수정 후 승인.
4. "@딥리서치"로 에이전트 호출 → 우패널 '활동' 탭에 서브에이전트 병렬 진행 표시 → 최종 보고서 아티팩트 + 인용.

## 2.2 무드 & 시그니처

**무드 키워드**: 정밀(precision) · 신뢰(trust) · 절제(restraint). 정밀기계 제조사의 도구다운 화면 — 얇은 1px 보더, 명확한 그리드, 넉넉하되 낭비 없는 여백, 장식 없는 라인 아이콘(lucide 계열). 글래스모피즘·네온 그라디언트·큰 그림자·일러스트 남용 금지. 모션은 기능적(150–200ms ease-out)이고 `prefers-reduced-motion` 존중.

**시그니처 요소 — 실행 레일(Run Rail)**: 에이전틱 턴(도구를 사용한 assistant 응답)의 좌측에 붙는 2px 수직 레일 + 눈금(tick). 공작기계의 리니어 가이드/DRO 눈금에서 가져온 은유로, 턴 안에서 발생한 이벤트(도구 시작·결과·HITL·인용·아티팩트)마다 눈금이 하나씩 새겨진다. 눈금 색 = 이벤트 상태(진행=primary 펄스, 완료=success, 오류=danger, 승인 대기=warning). hover 시 이벤트명 툴팁, 클릭 시 우패널 '활동' 탭의 해당 스텝으로 이동. 진행 중에는 레일이 위→아래로 채워진다. **이것이 WChat을 기억하게 만드는 단 하나의 브랜드 요소다** — 나머지는 조용하게. 수치(경과 시간, 토큰 수, 페이지 번호)는 모두 모노스페이스 tabular numerals로 표기해 계측기의 정밀함을 잇는다.

**레드 사용 규율**: 순수 #FF0000은 **로고 자산 안에서만** 존재한다. UI 기능적 빨강(오류·파괴적 액션)은 화면 가독을 위해 PANTONE 186C의 표준 sRGB 변환값 **#C8102E**를 사용한다(디지털 적응 결정 — CI의 인쇄 기준과 구분해 명시). 빨강을 대면적 배경·장식·활성 상태에 쓰지 않는다.

# 3. UX 원칙 10 (모든 프레임에 적용 — 2026 상용 에이전틱 챗 딥리서치 기준)

1. **행위 투명성 = 신뢰의 회계 계층.** 모든 도구 호출은 발화 위치에 인터리브된 카드로 표시. MCP 도구는 `커넥터명 › 도구명` 출처 라벨 필수. 무엇이 실행됐는지 숨기지 않는다.
2. **상태 어휘 하나.** queued(대기) / running(실행 중) / done(완료) / error(오류) / needs-approval(승인 필요) — 전 화면이 동일한 StatusChip으로 말한다. 색 매핑: running=primary 펄스, done=success, error=danger, needs-approval=warning.
3. **통제권 상시.** 스트리밍 중 Stop은 항상 도달 가능. HITL은 **부수효과(쓰기·삭제·외부 전송) 도구에만** 발동 — 읽기 전용은 무프롬프트(과승인은 자동화를 죽인다). HITL 카드에서 인자를 인라인 수정 후 승인 가능.
4. **점진 공개.** 추론(reasoning)은 접힌 "N초 생각" 칩이 기본. 도구 입력/출력의 대형 payload는 요약 + "더 보기". 설명은 요청 시 제공(explainability on demand) — 기술 로그를 쏟아붓지 않는다.
5. **인용 우선.** 지식·웹 검색 답변은 문장 끝 [N] 칩 + hover 스니펫 팝오버 + 클릭 시 우패널 '출처' 탭에서 원문 하이라이트 + 응답 하단 Reference 목록(`[1] 열관리모듈_시험성적서.pdf p.7`).
6. **지연 체감 설계.** 전송 즉시 optimistic 표시, 첫 토큰 전 shimmer. 강제 오토스크롤 금지 — 사용자가 위로 벗어나면 하단에 "최신으로 ↓" pill. 스트리밍 중 미완성 코드펜스는 닫힐 때까지 지연 렌더.
7. **능력 발견성.** 사용자가 무엇을 부를 수 있는지(커넥터·에이전트·도구)를 홈 제로 상태와 컴포저 @멘션에서 투명하게 노출. 빈 화면은 초대장이다.
8. **복구 경로 우선.** 오류는 원인별로: 재시도 가능한 것에만 [다시 시도], 429는 백오프 안내, 크레딧 부족은 재시도 없이 다음 행동 안내. 오프라인 배너 + 자동 재연결. 입력 드래프트는 항상 보존.
9. **키보드 우선 + 엔터프라이즈 밀도.** Cmd+K 커맨드 팔레트(세션·액션 검색), Cmd+Enter 전송, Cmd+\ 우패널 토글, Cmd+N 새 세션. 관리 테이블은 밀도 높게, 채팅은 여유 있게 — 이중 밀도.
10. **접근성 내장.** WCAG 2.2 AA 대비(위 토큰 페어 준수), 스트리밍 영역 `aria-live="polite"` + 디바운스 안내, HITL `aria-live="assertive"`, 새 턴이 포커스를 훔치지 않음, Tab만으로 전 흐름 도달.

**MentionMenu(@)** — 컴포저 위 팝오버 360px. 상단 검색 입력(자동 포커스), 탭 [전체|에이전트|도구|커넥터|파일|지식]. 행: 아이콘 + 이름 + 출처 보조라벨(내장 | MCP: 서버명 | 프로젝트명 | 이 대화) + 우측 정책 배지(`읽기 전용` neutral / `승인 필요` warning). 키보드 ↑↓ 하이라이트, Enter 삽입 — 삽입 결과는 컴포저 안 토큰 칩(primary-50, 제거 ×). 결과 없음: "일치하는 항목이 없습니다 — 커넥터 등록에서 추가할 수 있습니다" 링크.

**슬래시(/) 커맨드** — 동일 팝오버 축소형, 액션 리스트: /요약 · /표로 정리 · /브랜드PPT · /딥리서치 · /번역. 선택 시 컴포저에 프리필.

**ModelPicker** — 제공사 그룹 헤더(Anthropic/OpenAI/Google) + 행: 모델명 + capability 뱃지(도구·비전·추론) + 컨텍스트 `200K`(mono) + 조직 정책 라벨. 하단 구분선 후 추론 강도 세그먼트(낮음|보통|높음). 미허용 모델은 목록에서 숨김(회색 처리 아님).

**Composer** — 구조: (첨부 칩 행) → 텍스트영역(auto-grow ≤10줄, placeholder "메시지를 입력하세요 — @로 에이전트·도구 호출") → 액션바: 좌 [＋][@][/], 중 모델 칩·모드 토글·웹검색 토글, 우 컨텍스트 게이지(옵션, mono `12%`) + 전송(primary, 스트리밍 중엔 Stop으로 교체). 드래그 오버 시 전체 드롭존 하이라이트(primary-50 + 점선 보더) "여기에 놓아 첨부".

**Reasoning** — 스트리밍 중 "생각 중…" 인라인 요약 1줄, 완료 시 `12초 생각` 칩으로 접힘(neutral). 클릭 펼침.

**RunRail** — §2.2. 눈금 간격은 이벤트 순서 기반 균등, 최소 12px.

**ConnectorCard / AgentCard / AttachmentChip / MessageActions / Toast / EmptyState** — F10·F11·F4 스펙 준수. Toast 4종(info/success/warning/danger), 우상단, 자동 5초 + 액션 링크 허용. EmptyState: 라인 아이콘 + 한 줄 안내 + CTA 1개(장식 일러스트 금지).

# 7. 상태·모션·접근성 규격

- 로딩: 실제 레이아웃과 동일 골격의 Skeleton(animate-pulse). 스트리밍 첫 토큰 전: assistant 영역 shimmer 3줄.
- 오류 배너(턴 내): danger-50 배경 + 원인 문구 + 조건부 [다시 시도]. 429: "요청이 많습니다 — 32초 후 자동 재시도"(mono 카운트). 오프라인: 상단 고정 배너 + 자동 재연결 시도 표시.
- 전송 optimistic: 사용자 메시지 즉시 표시, 실패 시 메시지에 warning 아이콘 + [재전송].
- 모션: 150–200ms ease-out, 패널 슬라이드 240ms. 펄스·shimmer는 reduced-motion에서 정적 대체.
- 대비: 본문 neutral-900/neutral-0 ≥ 12:1, 보조 neutral-500 ≥ 5.7:1, primary-600 on white ≥ 10:1, 흰 텍스트 on primary-600 통과 — 이 페어만 사용.
- 스크린리더: 채팅 로그 `role="log"` + `aria-live="polite"` + `aria-atomic="false"` + 안내 디바운스, HITL `aria-live="assertive"`. 아이콘 버튼 전부 접근 가능한 이름.

# 8. 카피 가이드 & 실데이터

톤: 간결한 존댓말, 능동태, 버튼은 결과를 말한다(전송·승인·수정 후 승인·다시 시도 — "확인/OK" 금지). 오류는 사과 대신 원인+다음 행동. 시스템이 아닌 사용자의 언어(예: "웹훅 설정"이 아니라 "커넥터"). 도메인 예시는 자동차부품·공작기계·스마트팩토리로 한정(방산 도메인 예시는 쓰지 않는다).

세션 목록 예: 「등속조인트 공정 불량 원인 분석」 「열관리 모듈 시험성적서 요약」 「협력사 RFQ 회신 초안」 「공작기계 알람 코드 조회」 「스마트팩토리 PoC 회의록 정리」. 메모리 예: 「구동부품 품질팀 소속, 직무는 공정 데이터 분석」(자동 추출·📌) 「답변은 5문장 이내로 요약」(수동) 「사내 품질 가이드: wiki/qms-guide」(참조). 사용자명 김민수, 아바타 이니셜 "김".
